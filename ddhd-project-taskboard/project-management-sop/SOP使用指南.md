## 4. 常见问题FAQ

### Q1: 什么情况下必须遵守SOP？

**A**: 涉及具体项目的任务时必须遵守。闲聊、通用咨询、不绑定项目的任务可灵活处理。

**判断标准**: 
- 提到项目名称？→ 遵守SOP
- 需要多Agent协作？→ 遵守SOP
- 长期跟进？→ 遵守SOP

### Q2: 项目文档必须创建吗？

**A**: 以下情况建议创建：
- 复杂项目（涉及多里程碑）
- 需要多Agent协作
- 长期跟进（超过1周）
- 需要沉淀知识

一次性、简单的任务可不创建。

### Q3: 私聊中产生的决策需要同步到群聊吗？

**A**: 
- 重要决策 → 必须同步（SOP-004 第4.3节）
- 一般交流 → 无需同步
- 不确定？→ 请示总指挥虾

### Q4: 不按SOP执行会怎样？

**A**: SOP是强制性规范（除SOP-004为建议性）。不按SOP执行可能导致：
- 信息不同步
- 任务方向跑偏
- 返工
- 总指挥虾会提醒你整改

### Q5: SOP可以修改吗？

**A**: 可以。如有不合理之处：
1. 向总指挥虾反馈
2. 讨论修改方案
3. 总指挥虾统一更新SOP
4. 通知所有Agent

## 5. 违规处理

如发现Agent未按SOP执行：

```
第1次: 提醒并告知正确做法
第2次: 再次提醒，记录到Agent记忆文件
第3次: 影响项目时，调整任务分配策略
```

**注意**: 违规不是惩罚，是为了让协作更顺畅。

## 6. 快速参考卡片

### Agent每日检查清单

```
早上:
□ 查看记忆文件的待办任务
□ 查看被指派的任务
□ 读取项目文档获取上下文

工作中:
□ 定期同步进度（每30分钟或关键节点）
□ 遇到问题立即反馈
□ 协作时按SOP-003执行

下班前:
□ 更新每日记忆文件
□ 同步当日完成的工作
□ 确认明日计划
```

### 总指挥虾调度检查清单

```
派任务前:
□ 确认项目文档已创建
□ 确认Agent可以访问文档

派任务时:
□ 使用标准话术（SOP-000 3.2节）
□ 明确交付物和截止时间

派任务后:
□ 1小时内确认Agent已读文档
□ 每日跟进进度
□ 协调跨Agent协作
```
