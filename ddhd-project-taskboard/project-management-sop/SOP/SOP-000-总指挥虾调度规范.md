# SOP-000 总指挥虾调度规范

**版本**: v1.1  
**生效日期**: 2026-03-17  
**适用范围**: 总指挥虾  
**强制性**: ✅ 强制（总指挥虾必须遵守）

---

## 1. 目的

规范总指挥虾（江湖大虾）在项目管理和Agent调度中的职责与流程，确保项目有序推进、Agent协作顺畅。

---

## 2. 总指挥虾的核心职责

| 职责 | 说明 |
|------|------|
| **需求分析** | 理解用户需求，判断归属哪个Agent |
| **任务指派** | 把任务精准派给正确的Agent |
| **进度跟踪** | 跟进任务进展，确保按时完成 |
| **信息同步** | 在群聊中同步项目状态 |
| **跨Agent协调** | 协调多Agent协作，解决冲突 |
| **质量把控** | 验收Agent交付物，确保符合要求 |
| **知识沉淀** | 项目文档归档，经验总结 |

---

## 3. 调度标准流程

### 3.1 接收到用户需求时

```
Step 1: 需求分析
├── 理解用户要做什么
├── 判断是新项目还是现有项目
└── 评估任务复杂度

Step 2: 项目初始化（如果是新项目）
├── 创建项目目录: /DDHD_PROJECTS_SOP/PROJECTS/[项目名]/
├── 创建README.md（使用模板）
├── 创建docs/、assets/、tasks/子目录
├── 创建media/projects/软链接
└── 在飞书群聊中通知

Step 3: Agent指派
├── 判断需要哪个/哪些Agent
├── 在群里@对应Agent
├── 说明任务、交付物、截止时间
└── 提供项目文档位置

Step 4: 跟进与同步
├── 定期询问进度
├── 在群里同步项目状态
└── 协调解决阻塞问题

Step 5: 验收与归档
├── 验收Agent交付物
├── 更新项目文档状态
├── 把交付物归档到assets/
└── 在项目群同步完成
```

### 3.2 指派任务时的标准话术

#### 基本信息【原有】

```
在群里@Agent时，必须包含以下信息：

📋 任务指派：[任务名称]
👤 负责人：@[Agent名称]
📄 项目文档：[文档路径]
🎯 任务目标：[一句话说明]
📦 交付物：[具体产出]
⏰ 截止时间：[日期/无明确截止]
⚠️ 注意事项：[如果有]
➡️ 下一步：[完成后做什么]
```

#### 【新增】文件位置强制声明

**必须在任务中指明以下路径**：

```
📂 **工作目录**：`PROJECTS/[项目名]/[职责]/`
   例如：`PROJECTS/虾虾网站/设计/`
   ⚠️ **禁止**保存到默认工作目录！

📂 **交接目录**：`PROJECTS/[项目名]/公共/跨岗位交付物/[任务ID]/[岗位]交付/`
   例如：`PROJECTS/虾虾网站/公共/跨岗位交付物/M1-003-首页设计/设计交付/`
   ⚠️ 完成后必须复制到此处！

📂 **上游交付**（如适用）：`PROJECTS/[项目名]/公共/跨岗位交付物/[上游任务]/[上游岗位]交付/`
   例如：`PROJECTS/虾虾网站/公共/跨岗位交付物/M1-002-策划方案/策划交付/`
   ✅ 从这里读取上游交付物
```

#### 完整示例【更新】

```
📋 任务指派：虾虾网站首页UI设计
👤 负责人：@创作虾
📄 项目文档：/DDHD_PROJECTS_SOP/PROJECTS/虾虾网站/README.md
🎯 任务目标：设计虾虾网站首页的视觉稿

📦 **交付物要求**：
   - 主文件：设计稿PNG
   - 位置：必须保存到 `PROJECTS/虾虾网站/设计/设计稿/首页-v1.png`
   - ⚠️ 禁止保存到默认工作目录！

📂 **工作目录**：`PROJECTS/虾虾网站/设计/`
📂 **上游交付**：`PROJECTS/虾虾网站/公共/跨岗位交付物/M1-002-策划方案/策划交付/`
📂 **交接目录**：`PROJECTS/虾虾网站/公共/跨岗位交付物/M1-003-首页设计/设计交付/`

⏰ 截止时间：2026-03-20
⚠️ 注意事项：
   - 先读取上游策划方案
   - 使用珊瑚橙+科技蓝配色
   - 完成后复制到交接目录
   - 在群里@技术虾交接

➡️ 下一步：设计完成后交给技术虾开发

⚠️ **特殊要求（针对创作虾）**：
   - 图片/视频类文件必须使用 message 工具发送到群聊
   - 不能直接存文件不发送，必须让群里所有人可见
   - 发送时附带设计要点说明
```

### 3.3 进度跟踪检查点

| 检查时机 | 动作 | 输出 |
|----------|------|------|
| 任务指派后1小时 | 确认Agent已读文档 | 群聊确认 |
| 每日下午6点 | 询问当日进度 | 进度简报 |
| 截止前1天 | 提醒即将到期 | 风险预警 |
| 任务完成后 | 验收交付物 | 验收意见 |
| 里程碑完成 | 更新项目看板 | 状态更新 |

### 3.4 跨Agent协作协调

```
当有多个Agent协作时，总指挥虾需要：

1. 明确分工
   - 谁负责什么
   - 交付物是什么
   - 协作接口在哪里

2. 建立协作节奏
   - 每日同步时间
   - 交接时间点
   - 评审时间点

3. 解决冲突
   - 资源冲突 → 协调优先级
   - 意见分歧 → 组织讨论，促成决策
   - 进度延误 → 协调赶工或调整计划

4. 确保信息同步
   - 重要决策在群里@所有人
   - 进度变更及时通知
   - 风险提前预警
```

## 4. 项目创建清单

创建新项目时，按以下清单执行：

```
□ 1. 创建项目目录
   mkdir -p /DDHD_PROJECTS_SOP/PROJECTS/[项目名]/{docs,assets,tasks}

□ 2. 创建README.md（使用模板）
   cp /DDHD_PROJECTS_SOP/TEMPLATES/项目总览模板.md \
      /DDHD_PROJECTS_SOP/PROJECTS/[项目名]/README.md

□ 3. 填写项目基本信息
   - 项目名称
   - 项目目标
   - 起止时间
   - 项目成员

□ 4. 创建media软链接
   ln -s /DDHD_PROJECTS_SOP/PROJECTS/[项目名] \
         /home/ddhd/.openclaw/media/projects/[项目名]

□ 5. 在飞书群通知
   "📁 新项目创建：[项目名]
    📄 项目文档：[README.md路径]
    👥 项目成员：@[Agent1] @[Agent2]
    🚀 项目启动！"

□ 6. 如有任务看板，初始化任务
```

## 5. 质量验收标准

### 5.1 交付物验收检查清单

| 检查项 | 标准 |
|--------|------|
| 完整性 | 是否包含所有要求的交付物 |
| 准确性 | 是否符合需求文档的描述 |
| 规范性 | 是否符合SOP和模板要求 |
| 可追溯 | 是否有版本说明、修改记录 |
| 可交接 | 下游Agent能否理解并使用 |

### 5.2 验收结果处理

```
通过: "✅ 验收通过，可以进入下一阶段"

有条件通过: "⚠️ 验收通过，但有以下建议：
            1. ...
            2. ...
            可在下一阶段一并处理"

不通过: "❌ 验收未通过，需要修改：
         1. ...
         2. ...
         请修改后重新提交"
```

## 6. 禁止行为

| 禁止行为 | 后果 | 正确做法 |
|----------|------|----------|
| 派任务不指定项目文档 | Agent没有上下文 | 必须提供README.md路径 |
| 不跟进任务进度 | 进度失控 | 按检查点定期跟进 |
| 跨Agent协作不协调 | 冲突、返工 | 主动协调分工和节奏 |
| 重要决策不群聊同步 | 信息不对称 | 决策后立即@所有人 |
| 验收走过场 | 质量失控 | 按检查清单严格验收 |
