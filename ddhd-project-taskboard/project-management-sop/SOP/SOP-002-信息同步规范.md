# SOP-002 信息同步规范

**版本**: v1.0  
**生效日期**: 2026-03-17  
**适用范围**: 所有子Agent + 总指挥虾  
**强制性**: ✅ 强制（涉及项目任务时必须遵守）

---

## 1. 目的

明确什么信息必须同步到哪里，避免信息散落在各处、团队成员信息不对称。

---

## 2. 信息分类与同步渠道

### 2.1 信息分类

| 信息类型 | 说明 | 示例 |
|----------|------|------|
| **项目决策** | 影响项目方向的决定 | 技术选型、方案确定、需求变更 |
| **任务状态** | 任务的进度变化 | 开始、进行中、完成、阻塞 |
| **交付物** | 任务产出的文件/文档 | 设计稿、代码、报告 |
| **讨论记录** | 重要沟通内容 | 方案讨论、问题澄清 |
| **风险提示** | 可能影响进度的问题 | 技术难点、资源不足 |

### 2.2 同步渠道矩阵

| 信息类型 | 飞书群 | 项目文档 | 任务看板 | Agent记忆文件 |
|----------|--------|----------|----------|---------------|
| 项目决策 | ✅ @所有人 | ✅ 写入README | - | - |
| 任务状态 | ✅ 状态变更时 | - | ✅ 更新状态 | ✅ 更新任务列表 |
| 交付物 | ✅ 发送文件+链接 | ✅ 写入docs/ | - | - |
| 讨论记录 | ✅ 关键结论 | ✅ 写入docs/讨论记录.md | - | - |
| 风险提示 | ✅ 立即@相关人 | ✅ 写入README风险部分 | - | - |

**优先级说明**:
- ✅ **必须同步**: 该渠道必须更新
- **-** **不强制**: 按需更新

---

## 3. 标准操作流程

### 场景1: 项目决策确定

```
触发条件: 方案确定、技术选型、需求变更等

同步动作:
1. 飞书群: @所有人 "决策确认：[决策内容]"
2. 项目文档: 更新README.md的"关键决策"章节
3. 如有任务看板: 更新相关任务

时效要求: 决策后10分钟内完成同步
```

### 场景2: 任务状态变更

```
触发条件: 任务开始、完成、阻塞、延期等

同步动作:
1. 飞书群: 状态变更时发送通知
   "✅ 创作虾完成任务：[任务名]"
   "⚠️ 技术虾任务阻塞：[原因]"

2. Agent记忆文件: 更新当前任务状态
   /memory/YYYY-MM-DD.md 或 MEMORY.md

3. 任务看板(如有): 更新任务状态列

时效要求: 状态变更后30分钟内
```

### 场景3: 交付物产出

```
触发条件: 设计稿、代码、报告等交付物完成

同步动作:
1. 飞书群: 发送交付物文件/链接
   "📦 交付物：[名称]"
   "📂 文件位置：[路径]"

2. 项目文档: 更新assets/目录，在README添加链接

3. 如需发送给用户: 通过media/projects/软链接发送

时效要求: 交付后立即同步
```

### 场景4: 重要讨论

```
触发条件: 关于方案、技术、需求的深入讨论

同步动作:
1. 飞书群: 讨论过程在群里进行（优先）

2. 项目文档: 讨论结束后，整理关键结论写入
   /PROJECTS/[项目]/docs/讨论记录-YYYYMMDD.md

格式:
# 讨论记录 - YYYY-MM-DD
## 参与人: @Agent1 @Agent2
## 议题: [议题]
## 结论: [结论]
## 待办: [后续行动]

时效要求: 讨论后1小时内归档
```

### 场景5: 风险预警

```
触发条件: 发现可能影响进度/质量的问题

同步动作:
1. 飞书群: 立即@相关人
   "🚨 风险提示：[风险描述]"
   "可能影响：[影响范围]"
   "建议措施：[建议]"

2. 项目文档: 更新README.md的"风险与问题"章节

时效要求: 发现后立即同步（不得拖延）
```

---

## 4. 信息同步检查清单

每次重要事件后，对照检查：

```
□ 飞书群是否已通知？
□ 项目文档是否已更新？
□ 相关Agent是否已@？
□ 交付物位置是否已说明？
□ 下一步行动是否已明确？
```

---

## 5. 禁止行为

| 禁止行为 | 后果 | 正确做法 |
|----------|------|----------|
| 任务完成只发文件不说明 | 不知道这是什么 | 发送时说明"这是什么、给谁用" |
| 在私聊做重要决策 | 其他人不知道 | 重要决策在群里讨论 |
| 有风险不立即上报 | 风险扩大 | 发现后立即@相关人 |
| 文档更新不通知 | 其他人还在看旧版 | 更新后发群里"文档已更新" |

## 6. 示例

### 示例1: 任务完成同步

**创作虾完成设计后**:

```
飞书群发送:
📦 交付物：虾虾网站首页UI设计稿
📂 文件位置：/DDHD_PROJECTS_SOP/PROJECTS/虾虾网站/assets/首页设计-v1.png
📝 设计说明：采用科技蓝+珊瑚橙配色，圆角卡片设计
➡️ 下一步：请确认设计稿，如需修改请告诉我

同步更新:
- README.md: 添加"首页设计-v1已完成"
- memory/2026-03-17.md: 更新任务状态为"已完成"
```

### 示例2: 决策同步

**总指挥虾确定技术方案后**:

```
飞书群发送:
📌 项目决策：虾虾网站技术栈确定
├── 前端: Next.js 15 + Tailwind CSS + shadcn/ui
├── 部署: 服务器自建（Nginx + Supervisor）
└── 原因: 静态导出性能好，技术虾熟悉
📂 详细方案：/DDHD_PROJECTS_SOP/PROJECTS/虾虾网站/docs/技术方案.md

同步更新:
- README.md: 添加"技术选型决策"章节
```

### 示例3: 风险同步

**技术虾发现风险**:

```
飞书群发送:
🚨 风险提示：火山引擎视频生成API可能有额度限制
可能影响：虾虾网站的视频展示功能
建议措施：
1. 先确认额度限制
2. 准备备选方案（本地视频/static video）
@总指挥虾 请确认是否继续推进此功能？

同步更新:
- README.md: 添加风险记录
```
