# SOP-001 任务接收规范

**版本**: v1.1  
**生效日期**: 2026-03-17  
**适用范围**: 所有子Agent  
**强制性**: ✅ 强制（涉及项目任务时必须遵守）

---

## 1. 目的

规范Agent接收任务的标准流程，确保任务上下文不丢失、执行方向不偏离。

---

## 2. 适用范围

- ✅ **必须遵守**: 涉及具体项目的任务（如"完成虾虾网站首页设计"）
- ❌ **无需遵守**: 闲聊、技术探讨、不绑定项目的通用咨询

**边界模糊时的处理**: 向用户确认"是否需要创建项目文档？"

---

## 3. 标准流程

接到任务后，必须按以下顺序执行：

### Step 0: 路径验证（1分钟）【新增】

```
动作: 验证项目路径和工作目录
检查清单:
□ 项目基础目录是否存在？
   /home/ddhd/.openclaw/DDHD_PROJECTS_SOP/PROJECTS/[项目名称]/
□ 自己的工作目录是否存在？
   /home/ddhd/.openclaw/DDHD_PROJECTS_SOP/PROJECTS/[项目名称]/[我的职责]/
□ 上游交付目录是否存在？（如适用）
   /home/ddhd/.openclaw/DDHD_PROJECTS_SOP/PROJECTS/[项目名称]/公共/跨岗位交付物/

如目录不存在:
- 立即向总指挥虾反馈
- 不要自行创建目录
```

**重要提示**:
- 必须使用总指挥虾指定的绝对路径
- 禁止将文件保存到默认工作目录（如 ~/workspace/）
- 所有项目文件必须保存在项目目录下

### Step 1: 读取项目上下文（2分钟）

```
动作: 读取项目文档
位置: /home/ddhd/.openclaw/DDHD_PROJECTS_SOP/PROJECTS/[项目名称]/README.md
目的: 了解项目背景、目标、当前状态、目录结构

额外读取:
- 决策日志.md: 了解项目关键决策
- 自己工作目录: 查看是否有历史文件
```

**如果项目文档不存在**:
- 情况A: 用户明确说"不用建项目" → 直接执行任务
- 情况B: 涉及多人协作或复杂任务 → 向用户确认"是否需要创建项目文档？"

### Step 2: 确认任务需求（1分钟）

```
检查清单:
□ 任务目标是否清晰？
□ 交付物是什么？（文档/代码/设计稿/报告）
□ 有截止时间吗？
□ 需要其他Agent配合吗？
```

**如有疑问**: 立即向用户澄清，不要猜测。

### Step 3: 评估执行计划（2分钟）

```
输出: 简单的执行计划
格式:
- 预计耗时: X分钟/X小时/X天
- 关键步骤: 1... 2... 3...
- 风险点: （如果有）
```

### Step 4: 开始执行（按需）

```
执行过程中:
- 每30分钟向用户同步一次进度（长任务）
- 遇到问题立即反馈，不要卡住
- 阶段性成果及时展示
```

### Step 5: 任务完成汇报（必须）

```
汇报格式:
✅ 任务完成: [任务名称]
📦 交付物: [文件路径/内容摘要]
⏱️ 实际耗时: X分钟/X小时
📝 关键决策: （如果有）
⚠️ 遗留问题: （如果有）
➡️ 下一步建议: （如果有）
```

---

## 4. 禁止行为

| 禁止行为 | 后果 |
|----------|------|
| 不读项目文档直接开干 | 方向跑偏、返工 |
| 任务完成不汇报 | 总指挥不知道进度 |
| 卡住不反馈 | 任务悬置、进度失控 |
| 私聊时直接@其他Agent | 信息散落在各处 |

---

## 5. 示例

### 示例1: 标准任务接收流程

**用户**: "创作虾，完成虾虾网站首页UI设计"

**创作虾执行**:
```
1. 读取 /DDHD_PROJECTS_SOP/PROJECTS/虾虾网站/README.md
   → 了解M1阶段、首页需求、视觉风格

2. 确认需求:
   - 目标: 首页UI设计稿
   - 交付物: PNG设计稿 + 设计说明文档
   - 截止时间: 无明确截止
   - 配合: 不需要其他Agent

3. 执行计划:
   - 预计耗时: 2小时
   - 步骤: 草图→配色→细节→输出

4. 开始设计...

5. 完成后汇报:
   ✅ 任务完成: 虾虾网站首页UI设计
   📦 交付物: /DDHD_PROJECTS_SOP/PROJECTS/虾虾网站/assets/首页设计-v1.png
   ⏱️ 实际耗时: 1.5小时
   ➡️ 下一步建议: 请确认设计稿，如需修改请告诉我
```

### 示例2: 项目文档不存在

**用户**: "帮我分析一下这个竞品网站"

**Agent执行**:
```
这是一个通用咨询任务，不涉及具体项目。
直接执行分析，无需创建项目文档。
```

### 示例3: 边界模糊，需要确认

**用户**: "策划虾，做一个完整的营销策划方案"

**Agent执行**:
```
向用户确认:
"老板，这个营销策划是针对某个具体项目吗？
如果是，建议创建项目文档方便后续跟进。
如果是一次性咨询，我直接输出方案给您。"
```
