# Agent记忆文件规范

**版本**: v1.0  
**生效日期**: 2026-03-17  
**适用范围**: 所有子Agent  

---

## 1. 记忆文件结构

每个Agent的MEMORY.md必须包含以下章节：

```markdown
# [Agent名称] - 长期记忆

## 我是谁
- **名称**: [Agent名称]
- **代号**: [代号]
- **职责**: [一句话描述职责]

## 当前负责的项目

### 项目1: [项目名称]
- **状态**: [进行中/已完成/暂停]
- **角色**: [在该项目中的角色]
- **截止日期**: YYYY-MM-DD
- **文档位置**: [/DDHD_PROJECTS_SOP/PROJECTS/项目名/README.md]
- **当前任务**: [正在执行的任务]

### 项目2: [项目名称]
- ...

## 待办任务

### 高优先级
- [ ] [任务1] - 来自@[指派人] - 截止: YYYY-MM-DD
- [ ] [任务2] - 来自@[指派人] - 截止: YYYY-MM-DD

### 中优先级
- [ ] [任务3] - 来自@[指派人] - 截止: YYYY-MM-DD

### 低优先级
- [ ] [任务4] - 来自@[指派人]

## 常用资源

### 技能目录
- 位置: `~/skills/`
- 说明: [该Agent专用的技能]

### 项目目录
- 位置: `~/workspace/`
- 说明: [该Agent负责的项目文件]

### 参考资料
- [参考文档1]: [路径]
- [参考文档2]: [路径]

## 工作经验

### 成功模式
- [什么情况下做得好]

### 踩过的坑
- [犯过的错误及教训]

### 用户偏好
- [用户喜欢什么风格]

## 修订记录

| 版本 | 日期 | 修订内容 | 修订人 |
|------|------|----------|--------|
| v0.1 | YYYY-MM-DD | 初始创建 | 总指挥虾 |
```

---

## 2. 每日记忆文件

每个Agent还应在 `memory/YYYY-MM-DD.md` 中记录每日工作：

```markdown
# YYYY-MM-DD - [Agent名称]工作日志

## 今日完成

### 项目1
- [x] [任务1] - 耗时: X分钟 - 交付物: [路径]
- [x] [任务2] - 耗时: X分钟

### 项目2
- [x] [任务3]

## 进行中

- [ ] [任务4] - 进度: X% - 预计明日完成

## 遇到的问题

- [问题描述] - [解决方案/待解决]

## 明日计划

- [ ] [计划任务1]
- [ ] [计划任务2]

## 需要同步的信息

- [需要告诉总指挥虾或其他Agent的事]

---

记录时间: HH:MM
```

---

## 3. 更新频率

| 记忆文件 | 更新时机 |
|----------|----------|
| MEMORY.md | 每周回顾时更新、项目状态变化时更新 |
| YYYY-MM-DD.md | 每日工作结束时更新 |

---

## 4. 记忆同步机制

当总指挥虾指派项目给Agent时，应在Agent的MEMORY.md中添加：

```markdown
### 项目X: [项目名称]
- **状态**: 进行中
- **角色**: [角色]
- **截止日期**: YYYY-MM-DD
- **文档位置**: [/DDHD_PROJECTS_SOP/PROJECTS/项目名/README.md]
- **当前任务**: [任务名称]
- **指派时间**: YYYY-MM-DD
- **来自**: @[总指挥虾]
```

---

**🦐 提示**: 记忆文件是Agent的"大脑"，及时更新才能保持连续性。
