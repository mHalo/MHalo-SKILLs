## 2. 目录结构说明

```
[项目名]/
├── README.md              ← 本文件（所有人必读）
├── 决策日志.md             ← 关键决策（所有人可读）
├── 里程碑进度.md           ← 进度看板（所有人查看）
│
├── 策划/                   ← 策划专属工作区（策划虾使用）
│   ├── 需求分析/
│   ├── 竞品分析/
│   └── 策划方案/
│
├── 设计/                   ← 设计专属工作区（创作虾使用）
│   ├── 设计稿/
│   ├── 素材/
│   └── 切图/
│
├── 开发/                   ← 开发专属工作区（技术虾使用）
│   ├── 技术方案/
│   ├── API文档/
│   └── 部署脚本/
│
├── 运营/                   ← 运营专属工作区（运营虾使用）
│   ├── 运营策略/
│   ├── 数据分析/
│   └── 内容规划/
│
└── 公共/                   ← 共享协作区
    ├── 会议记录/
    ├── 讨论归档/
    └── 跨岗位交付物/       ← 按任务划分的交接区
        ├── [任务ID]-[任务名]/
        │   ├── 策划交付/
        │   ├── 设计交付/
        │   └── 开发交付/
        └── ...
```

## 4. 跨岗位协作流程

### 交接示例

```
Step 1: 策划完成需求
   策划虾在自己的工作区写方案
   → 策划/策划方案/首页需求.md

Step 2: 策划交付给设计
   复制到交接区
   → 公共/跨岗位交付物/M1-003-首页设计/策划交付/首页需求.md
   
   在群里@创作虾：
   "@创作虾 首页策划已完成，文件在
    公共/跨岗位交付物/M1-003-首页设计/策划交付/"

Step 3: 设计接收并执行
   创作虾读取交接区文件
   → 公共/跨岗位交付物/M1-003-首页设计/策划交付/
   
   在自己工作区设计
   → 设计/设计稿/首页-v1.png

Step 4: 设计交付给开发
   复制到交接区
   → 公共/跨岗位交付物/M1-003-首页设计/设计交付/首页-v1.png
   
   在群里@技术虾：
   "@技术虾 首页设计已完成，文件在
    公共/跨岗位交付物/M1-003-首页设计/设计交付/"

Step 5: 开发接收并执行
   技术虾读取交接区文件
   → 公共/跨岗位交付物/M1-003-首页设计/设计交付/
   
   在自己工作区开发
   → 开发/代码/...
```
